# 本仓库说明

## 仓库性质

本仓库是 **飞书文档到 Dify 知识库自动化管线的维护仓库**。

它的职责有两个：

1. 维护这条自动化管线相关模块的源代码（含可选子包、工具，例如根目录下 `onboard/` 中的 `feishu-onboard`）。
2. 创建并初始化用于执行该管线的 Cursor 工作区。

这里的“初始化”包括但不限于：

- 工具接入与配置
- `AGENTS.md` 模板
- `rules` 规则文件
- 其他运行该管线所需的工作区上下文
- **入轨用辅助工具**（人触发的**非** webhook 主链路进程）：`onboard/` 提供可安装的 `feishu-onboard` CLI，按设计创建飞书 App 文件夹、尝试企业内可见、**两阶段**将业务映射与索引写入**仓库根** `.env`（与 `DIFY_TARGET_*` 等合同一致，详见仓库内 `docs/superpowers/specs/2026-04-26-feishu-app-folder-onboard-design.md`），并在输入完成后在目标工作区执行 `lark-cli` 初始化

## 不要混淆

本仓库 **不是** 一次具体同步任务的运行时工作区。

本仓库维护的是：

- 管线源码（含 `onboard/` 入轨子包，供运维或初始化使用）
- 工作区模板
- 工作区初始化逻辑
- 提示词与规则资产

本仓库不等于：

- 飞书 webhook 运行实例
- RQ worker 运行实例
- 某一次被启动的 Cursor CLI 会话
- 某一个已经生成好的业务执行工作区

换句话说：

- 这里负责“造系统、维护系统”。
- 真正执行单次同步任务的，是由外部模块启动、并由本仓库生成或初始化出来的 Cursor 工作区。

## 当前业务链路参考

本节用于提供业务理解与设计参考，帮助判断本仓库在整体链路中的位置。

这里描述的是 **当前已知的参考链路**，不是最终定稿的运行流程，也不是必须一字不差照搬的实现规范。

当前参考链路大致如下：

1. 飞书指定云文件夹下的文档发生新增或修改。
2. 飞书开放平台通过 webhook 推送事件。
3. webhook 模块去抖后把任务送入 RQ 队列。
4. RQ 在指定工作区启动 Cursor CLI，并注入任务上下文，例如飞书事件 ID、文档 ID 等。
5. 工作区中的 `AGENTS.md` 与规则文件通过 Cursor 机制为代理提供执行约束。
6. 代理调用工具获取飞书文档内容，并执行后续同步动作。
7. 代理调用 Dify 上传工具完成入库。

其中目前已知的是：

- webhook 模块在别处实现。
- 去抖与 RQ 队列在别处实现。
- 飞书获取工具在别处实现。
- 本仓库需要维护与这些模块协作的源码、配置、提示词和工作区初始化逻辑。

**`feishu-onboard` 不在上述 1–7 步之内**：它不接收 webhook 推送，也不替代单次任务里的 Agent；只在需要为新 `route_key` 建 App 文件夹、写回根 `.env` 与补索引时由人在维护仓或目标执行工作区根目录下运行。

如果后续业务编排、模块职责或执行顺序发生调整，应以新的设计与实现为准，而不是把本节当成冻结不变的流程定义。

## 本仓库的边界

在本仓库中工作时，优先把问题归类为以下两类之一：

### 1. 管线维护问题

例如：

- 管线模块源码如何组织
- 与外部 webhook / RQ / 工具的接口如何对接
- Dify 上传工具如何维护
- 配置、模板、规则如何版本化
- `onboard`（`feishu-onboard`）的接口、`.env` 键名与 spec §5/§6 对齐、与根 `.env` 合同联动的发布说明

### 2. 工作区初始化问题

例如：

- 如何创建 Cursor 工作区
- 如何按配置写入 `AGENTS.md`
- 如何分发或生成 `rules`
- 如何注入工具定义、上下文模板和运行约束

如果某个任务属于“单次文档同步如何执行”，要先判断它是在修改：

- 本仓库里的模板、规则或初始化逻辑

还是在处理：

- 某个已经生成好的执行工作区中的一次运行任务

这两者不能混为一谈。

## 对代理的要求

在本仓库内工作时，默认遵守以下原则：

- 先判断当前修改是在维护仓库本身，还是在描述生成后的工作区行为。
- 写规则时，明确区分“仓库级说明”和“运行时工作区说明”。
- 不要把外部已实现模块误写成本仓库内部实现。
- 不要把一次任务运行态的上下文，误当成仓库长期静态配置。
- 若边界不清，先补充说明，再继续设计或改动。

## 后续提示词整理原则

后续提示词体系按以下方向收敛：

- 目标工作区的 `AGENTS.md` 模板内容保存在 `prompts/AGENTS.txt`
- 目标工作区的 `rules` 模板放在 `prompts/rules/`

这些文件是 **待注入真实工作区的长期模板资产**，先保存在本仓库的 `prompts/` 下，由后续初始化逻辑分发到实际工作区。

`task_prompt.md` 属于 webhook 侧运行时注入物，由对端模块维护，不在本仓库内作为模板资产保存。
