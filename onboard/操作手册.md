# feishu-onboard 使用说明

本工具帮你在飞书里新建一个「业务用的云盘文件夹」，并把相关设置写进仓库根目录的配置文件里，同时在本机完成飞书命令行登录配置。

下面按顺序做即可。每一步都写了**你要做什么**和**做完这步是为了什么**。下面出现的 **`C:\你的项目\NewVLA`** 请改成你本机真实的项目根路径（含空格时保留引号）。

**若用 Windows PowerShell 5.1**：同一条行里要顺序执行多条命令时，用**分号** `;` 连接；**不要**用 bash 里的 `` `&&` ``（会报错）。

---

## 第一步：确认本机已装好工具

**操作：** 用 **Windows PowerShell** 安装本工具时，可照抄下面三行，只改你的路径后执行（第一行是进入本仓库里的 `onboard` 子文件夹）：

```powershell
Set-Location "C:\你的项目\NewVLA\onboard"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e ".[test]"
```

若之后激活虚拟环境时提示**禁止运行脚本**，在**以管理员身份**打开的 PowerShell 里执行一次下面这句，关窗口后一般不用再执行：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**可选（维护人员要求「先跑通自检」时）**，在同一 `onboard` 目录下执行：

```powershell
Set-Location "C:\你的项目\NewVLA\onboard"
.\.venv\Scripts\python.exe -m pytest tests -v
```

**目的：** 让电脑能运行 `feishu-onboard` 这条命令。

---

## 第二步：确认改的是「哪一个项目文件夹」

**操作：** 先**切到项目根目录**（这一层里应有 `.env`；规则模板常在 `prompts/rules/`，不必在本仓库再建一份 `rules/` 复制体）：

```powershell
Set-Location "C:\你的项目\NewVLA"
```

若维护人员说「工具找不到项目根」，在**同一个 PowerShell 窗口**里、运行入轨**之前**再执行（关掉窗口后下次要重设；路径用**绝对路径**）：

```powershell
$env:FEISHU_ONBOARD_REPO_ROOT = "C:\你的项目\NewVLA"
```

**目的：** 工具只认这一层里的配置和规则文件，走错目录会白跑或改错文件。

---

## 第三步：检查根目录里的配置文件

**操作：** 用资源管理器或 Cursor 等编辑器打开**项目根**下的 `.env`，确认：飞书应用的 ID 和密码已填、你要用的那一套 Dify 相关项由维护人员文档约定且**都填全、不要留空**；并确认已填 **`FEISHU_ONBOARD_FOLDER_DELEGATE_OPEN_ID`**=分享委托人的 [open_id](https://open.feishu.cn)（一般为 `ou_` 开头），否则入轨会拒绝运行（本工具不再用「可选」二路径，见 `onboard/README.md`）。

若要用 PowerShell **往 `.env` 末尾追加一行**（把键和值换成你的，慎用以免重复键）：

```powershell
Set-Location "C:\你的项目\NewVLA"
Add-Content -Path ".\.env" -Value "SOME_KEY=某值" -Encoding utf8
```

用记事本手改时，保存请选 **UTF-8** 编码，避免中文乱码。

**目的：** 没有这些，工具没法连飞书、也没法把「连哪个知识库」写正确。

---

## 第四步：准备好规则说明文件

**操作：** 在仓库里确认**已经存在**你要用的那条规则文件（维护本仓库时一般填模板路径，**不要**为了入轨再复制到 `rules/`）。相对路径只允许两种开头（不要用 `..`、不要绝对路径）：

- 执行工作区里物化后的：`rules/...`
- 本维护仓里直接指模板：`prompts/rules/...`（例如 `prompts/rules/qa/folders/folder_rule_template.mdc`）

记下你选的那条路径，等会儿**原样输入**（含大小写、正斜杠）。

可选：在项目根用 PowerShell **检查文件在不在**（把路径换成你的）：

```powershell
Set-Location "C:\你的项目\NewVLA"
Test-Path ".\prompts\rules\qa\folders\folder_rule_template.mdc"
```

若输出为 `True` 再往下做。

**目的：** 工具会检查这个文件真的存在，避免写错成不存在的文件。

---

## 第五步：确认本机有飞书命令行

**操作：**

```powershell
lark-cli --help
```

能正常出帮助即表示本机**能用到** `lark-cli`；若提示找不到命令，需先按飞书/团队文档安装，装好后再**新开**一个 PowerShell 重试。

**目的：** 入轨过程会在本机配好飞书命令行；没有它就跑不完「全流程成功」。

**与正文抓取同根：** `feishu-onboard` 里对 lark 做 `config init` / 校验时传入的**工作目录（cwd）**，须与**管线工作区根**一致，也就是根目录 `.env` 所在的那一层。之后在同一工作区里跑 `feishu_fetch`（或 Agent 任务触发的抓取）时，进程也应以**同一目录为当前工作目录**，或显式用环境变量让加载器指到**同一份**根 `.env`。在 A 目录入轨、在 B 目录跑抓取，会出现 `FEISHU_APP_ID` 与 lark 本地配置不一致等错误。

---

## 第六步：运行工具

**操作：** 每次运行前，先**切到项目根**并**激活**第一步里装好的虚拟环境（看到提示符前有 `(.venv)` 即表示已激活）：

```powershell
Set-Location "C:\你的项目\NewVLA"
.\onboard\.venv\Scripts\Activate.ps1
```

然后任选其一：

- 正常交互入轨：

```powershell
feishu-onboard
```

- 已有一条路线、但要**再新建一个**飞书文件夹时（维护人员会提醒你用这条）：

```powershell
feishu-onboard --force-new-folder
```

- 先看有哪些参数：

```powershell
feishu-onboard --help
```

若**不想**每次激活环境，可改用虚拟环境里的程序直接开跑（把路径改成你本机实际路径；若 `Scripts` 下没有 `.exe` 只有无后缀的 `feishu-onboard`，就改成那个文件名）：

```powershell
Set-Location "C:\你的项目\NewVLA"
& "C:\你的项目\NewVLA\onboard\.venv\Scripts\feishu-onboard.exe"
```

**目的：** 进入问答式录入，把下面各信息交给工具。

**说明：** 程序会先提示：新文件夹**默认在租户云空间根**创建；问「父文件夹」时**一直回车即可**，无需也不会要求你去抄父级 `token`。

---

## 第七步：按屏幕提示回答问题

**操作：** 出现英文提示时，依次输入。每一项的含义如下（你只需知道**填什么**）。

| 提示的大致意思 | 你填什么 | 这步在帮你做什么 |
|----------------|----------|------------------|
| 路线简称 | 英文大写简称，可含下划线，例如 `BIZ_A`；第一个字必须是**英文字母** | 给这条业务线起一个不会和别人混在一起的代号 |
| 文件夹名称 | 在飞书云盘里显示的名字，想叫什么填什么；空着可能有默认名 | 在飞书里新建文件夹时用的标题 |
| Dify 目标键 | 和 `.env` 里那套 Dify 配置**对应上**的英文大写名（问维护人员） | 让工具知道用哪一份知识库/接口设置 |
| 数据集编号 | 知识库里具体那一个数据集的 ID（问维护人员或从后台复制） | 把「文档要进哪」记进配置 |
| 规则文件路径 | 第四步记下的路径，如 `rules/qa/某某.md` 或 `prompts/rules/qa/某某.mdc` | 把「用哪条问答规则」记进配置 |
| 父文件夹编号 | **本管线默认不填**（直接回车）。表示在**当前企业/租户**的云空间**根下**建共享文件夹，不要求、也不需要父级。只有极少数排障/特殊目录需求才手填 `folder_token` | 不填=根目录；填了=在该父级下建子夹 |

**目的：** 把「哪条业务、哪个文件夹、哪套 Dify、哪条规则」一次性记清楚并写入配置。

**说明：** 如果这条「路线」以前已经跑过、配置里**已经有**这个文件夹的编号，你这次又不想建新文件夹，那么：**路线简称、Dify 目标键、数据集编号、规则路径**须和原来一致。想**再建一个新文件夹**时，**关闭当前运行**，回到**第六步**用带 `force` 的那条命令重新执行。

---

## 第八步：看是否跑成功

**操作：** 看最后屏幕上的提示和**退出码**（有的界面会显示数字 0、1、2、3；没有数字就以文字提示为准）。

| 你看到的 | 表示什么 | 你接下来要做什么 |
|----------|----------|------------------|
| 成功之类提示，且退出为 0 | 飞书里的文件夹、本机命令行、以及「登记到总目录」都做完 | 可请维护人员帮你核对配置文件里是否多了一条；也可自己在飞书云盘里看新文件夹 |
| 失败之类提示，退出为 2 | 有东西没配好，或和已有配置对不上，或连飞书失败等 | 把完整报错**脱敏**后发给维护人员（不要发应用密码、API 密钥、整段长日志里的敏感字） |
| 部分完成之类提示，退出为 3 | 文件夹可能已建、部分设置已写，但**给分享委托人加协作者**未成功、或本机 `lark-cli` 没完全成功 | 记下**文件夹链接**或编号，按 stderr 与维护人员排障，或再执行一次补全 |
| 中断、异常，退出为 1 | 例如你按了取消，或出现未预料错误 | 重试；仍不行把现象告诉维护人员 |

**说明（入轨、分享、权限）：** 工具**只**走一条路径：为根 `.env` 里 `FEISHU_ONBOARD_FOLDER_DELEGATE_OPEN_ID` 所指的成员调用飞书云空间**文件夹**协作者接口（`POST .../members?type=folder`），**不**调 `PATCH public`。成功**仅**表示该用户已被加成协作者；**组织内/全员在飞书内可见**仍须**该用户本人**在飞书客户端打开该文件夹，在「分享」里把可见范围调为组织内/全员等。若见 **1063002** 等，可按[开放平台《云文档常见问题》3](https://open.feishu.cn/document/ukTMukTMukTM/uczNzUjL3czM14yN3MTN#16c6475a) 在文件夹侧为**当前应用**做「**添加文档应用**」等。更多见 `onboard/README.md`「限流与错误码」下「分享委托人协作者」。

**说明（先单独联调、不入轨）：** 维护人员可让你在项目根执行 `feishu-onboard verify-delegate --open-id "ou_…"`（需 `.env` 中已有应用 ID/密钥及委托人相关键）。会新建**临时**文件夹并测加协作者，不写 `FEISHU_FOLDER_*` 业务映射；参数见 `onboard/README.md`「5.1 联调子命令」。

**目的：** 知道当前是**全部搞定**、**要补一步**，还是**要从头查配置**。

---

## 第九步：自己简单对一下

**操作：**

- 在飞书里打开上一步或屏幕里给到的链接，看新文件夹在不在、位置对不对。
- 让维护人员确认根目录配置里，已经出现和你「路线简称」对应的几项内容；若他提到「总名单」里有你的简称，即表示被正式登记。

**目的：** 避免以为成功了其实配置没写全。

---

## 安全提醒（必看）

- 不要对着镜头或他人屏幕展示应用密码、接口密钥、长串令牌。
- 发截图、录屏、聊天记录前，让维护人员教你怎么打码或裁掉敏感行。

若某一步不懂「该填什么」，以**负责环境的同事**或**本仓库说明文档**为准，不要猜填。
